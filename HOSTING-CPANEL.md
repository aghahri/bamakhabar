# راهنمای اجرای باماخبر روی هاست (cPanel)

## نکته مهم

بیشتر هاست‌های اشتراکی cPanel فقط **PHP** و **MySQL** دارند و **Node.js** را به‌صورت پیش‌فرض اجرا نمی‌کنند. پروژه باماخبر با **Next.js (Node.js)** ساخته شده.

پس دو حالت دارید:

---

## حالت ۱: cPanel شما «Node.js» دارد

در cPanel دنبال یکی از این‌ها بگردید:
- **Setup Node.js App** یا **Application Manager**
- یا در بخش **Software** گزینه مربوط به Node.js

اگر چنین گزینه‌ای بود، می‌شود همین برنامه را روی همان هاست اجرا کرد.

### چیزهایی که از هاست/cPanel لازم دارید

| مورد | توضیح |
|------|--------|
| **نسخه Node** | حداقل Node 18 (ترجیحاً 20). در بخش Node.js App معمولاً قابل انتخاب است. |
| **دیتابیس** | اگر **PostgreSQL** دارید: host، نام دیتابیس، کاربر، رمز، پورت. اگر فقط **MySQL** دارید باید در پروژه به MySQL سوئیچ کنیم. |
| **دامنه/ساب‌دامین** | مثلاً `bamakhabar.com` یا `www.bamakhabar.com` که به همین اپلیکیشن اشاره کند. |
| **دسترسی SSH** (اختیاری ولی مفید) | برای اجرای دستورات در سرور (clone، npm، build، اجرا با pm2). |

### اتصال از گیت‌هاب و اجرا

۱. از طریق **SSH** به سرور وصل شوید (یا از File Manager + Terminal در cPanel).
۲. در پوشه‌ای که می‌خواهید پروژه باشد:
   ```bash
   git clone https://github.com/USERNAME/bamakhabar.git
   cd bamakhabar
   ```
۳. فایل **متغیرهای محیطی** بسازید (مثلاً `.env`):
   ```env
   DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/DATABASE"
   ```
   (اگر دیتابیس روی همان سرور است از `localhost` استفاده کنید؛ وگرنه آدرس سرور دیتابیس را بگذارید.)
۴. نصب و بیلد:
   ```bash
   npm install
   npx prisma generate
   npx prisma db push
   npm run db:seed
   npm run build
   ```
۵. اجرا:
   - اگر cPanel گزینه **Start Application** برای Node دارد، همان را استفاده کنید (معمولاً دستور start را خودش می‌گذارد).
   - وگرنه با **PM2** یا **node**:
     ```bash
     npm start
     ```
     یا با PM2 برای همیشه‌بالا ماندن:
     ```bash
     npx pm2 start npm --name bamakhabar -- start
     npx pm2 save
     npx pm2 startup
     ```
۶. در cPanel دامنه/ساب‌دامین را به پورت این اپ (مثلاً 3000) یا به پوشهٔ پروژه وصل کنید (بسته به اینکه پنل چطور پیکربندی می‌کند).

---

## حالت ۲: cPanel شما فقط PHP + MySQL دارد (بدون Node.js)

در این حالت **نمی‌شود** همین Next.js را مستقیم روی آن هاست اجرا کرد.

دو راه دارید:

### گزینه الف: هاست/سرور دیگری برای برنامه

- **برنامه** را روی جایی که Node پشتیبانی می‌کند اجرا کنید، مثلاً:
  - **Render** (همان کاری که الان دارید)
  - یا یک **VPS** با Node نصب‌شده
- **دیتابیس** را می‌توانید روی همان سرویس (مثلاً Render PostgreSQL) نگه دارید، یا اگر هاست cPanel شما **MySQL ریموت** می‌دهد، می‌شود پروژه را به MySQL تغییر داد و دیتابیس را روی همان cPanel استفاده کرد.

### گزینه ب: دیتابیس روی cPanel، برنامه روی جای دیگر

- در cPanel یک **MySQL** (یا اگر داشت PostgreSQL) بسازید.
- اگر **دسترسی ریموت به MySQL** دارید: host، user، password، نام دیتابیس و پورت را به‌عنوان `DATABASE_URL` در جایی که برنامه را اجرا می‌کنید (مثلاً Render یا VPS) قرار دهید.
- برنامه را همان‌جا از گیت‌هاب اجرا کنید و فقط رشته اتصال دیتابیس را به هاست خودتان بدهید.

---

## خلاصه: از هاست چه چیزی را باید «بدهی» برای کانفیگ؟

برای **کانفیگ کردن هاست و اجرای برنامه** این‌ها را مشخص کن:

1. **آیا Node.js روی هاست هست؟** (بله/خیر)
2. **دیتابیس:** نوع (MySQL یا PostgreSQL) + آدرس هاست، نام دیتابیس، کاربر، رمز، پورت.
3. **دامنه:** مثلاً `bamakhabar.com` یا `www.bamakhabar.com`.
4. **دسترسی:** SSH (بله/خیر) یا فقط cPanel و File Manager.

اگر بگی هاستت Node دارد یا نه و دیتابیسش MySQL است یا PostgreSQL، می‌توان دقیق‌تر گفت چه مقدار از کانفیگ روی هاست است و چه مقدار روی گیت‌هاب/سرور دیگر.

---

## اجرا از گیت‌هاب با دیتابیس روی هاست

بله، می‌شود برنامه را از گیت‌هاب اجرا کرد و دیتابیس را روی هاست خودت نگه داشتی:

- روی **سرور/هاستی که Node دارد**: پروژه را با `git clone` از گیت‌هاب بگیر، سپس همان مراحل بالا (npm install، prisma، build، start) را با `DATABASE_URL` که به دیتابیس روی هاست اشاره می‌کند انجام بده.
- یا برنامه روی **Render** باشد و فقط `DATABASE_URL` را به دیتابیسی که روی cPanel ساختی (اگر ریموت دسترسی دارد) وصل کنی.

در هر دو حالت، کد از گیت‌هاب می‌آید و دیتابیس روی هاست (یا هر جای دیگری که connection string آن را بدهی) استفاده می‌شود.
